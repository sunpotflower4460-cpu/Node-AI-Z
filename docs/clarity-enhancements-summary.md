# システム理解強化の実装概要

## 問題の背景

Node-AI-Zは高度なAI実験アプリケーションですが、技術的な専門用語が多く、詳しくない人にとって「何が起きているのか」を正確に把握することが困難でした。

## 実装した改善内容

### 1. 共通UIコンポーネントの追加 (`src/ui/components/CommonUI.tsx`)

#### 新規コンポーネント

- **Tooltip**: マウスホバー/タップで説明を表示するツールチップ
- **HelpIcon**: 疑問符アイコンとツールチップを組み合わせたヘルプアイコン
- **StatusIndicator**: システムの状態（処理中/アイドル/アクティブ）を視覚的に表示

これらのコンポーネントにより、技術用語やUIの各要素に対して、文脈に応じた説明を提供できるようになりました。

### 2. メインページの強化 (`src/app/NodeStudioPage.tsx`)

#### 追加された説明

- **アプリケーション全体の説明**: Node-AI-Zが何をするアプリなのか、ヘッダーにヘルプアイコンを追加
- **実装方式の選択**:
  - 「じぶん会議方式」と「結晶思考方式」の違いを説明するツールチップ
  - 各ボタンに`title`属性で詳細説明
- **Surface Provider**: 表面的な返答生成の役割を説明
- **Internal reasoning**: 内部推論エンジンの共通性を説明
- **処理状態インジケーター**: 送信中/待機中の状態を視覚的に表示

### 3. セッション脳タブの強化 (`src/ui/tabs/SessionBrainTab.tsx`)

#### 初心者向け概要セクション

結晶思考方式の「セッション脳」という概念を、わかりやすい日本語で説明する導入セクションを追加：

- セッション脳とは何か
- なぜ重要なのか
- どのように使うのか

#### 技術用語の日本語化と説明

すべての技術的な項目に対してヘルプアイコンを追加し、詳細な説明を提供：

- **Session ID**: セッションの識別子
- **Turn Count**: 会話の回数（AIの経験値）
- **Afterglow**: 前回の会話の余韻・残響
- **Recent Activity**: 最近の会話の活発度
- **Field Intensity**: 会話の場の強さ
- **Temporal States**: 時間的特徴の履歴
- **Micro Signal Dimensions**: 微細な信号（空気感の数値化）
- **Prediction State**: 次の会話への予測

各項目は：
- 日本語のタイトル
- わかりやすい説明文
- ツールチップによる詳細な補足

### 4. 体験モードの強化 (`src/ui/modes/ExperienceMode.tsx`)

#### 追加された説明要素

- **モードの説明**: 体験モードが何をするモードかをヘルプアイコンで説明
- **Surface Provider**: AIプロバイダーの役割説明
- **Runtime選択**: Node/Signalの違いを説明
- **初回使用時のガイド**:
  - メッセージ入力の促し
  - AIの動作説明
  - ヒントセクションの追加

### 5. 観察モードの強化 (`src/ui/modes/ObserveMode.tsx`)

#### 追加された説明要素

- **観察研究モードの説明**: モードの目的と機能
- **ワークフローの説明**: 3ステップの使い方ガイド
- **機能バッジへのヘルプ**:
  - 「入力して分析」の説明
  - 「タブで内部を見る」の説明
  - 「履歴から見返せる」の説明

## 実装の特徴

### ユーザビリティ重視の設計

1. **非侵襲的**: ヘルプアイコンは小さく控えめで、通常の操作を邪魔しない
2. **オンデマンド**: 必要な時だけ情報にアクセスできる
3. **階層的**: 概要から詳細まで、段階的に理解を深められる

### アクセシビリティ

- `title`属性による標準的なツールチップ
- `aria-label`による支援技術への対応
- モバイル対応（タップでも表示可能）

### 視覚的フィードバック

- StatusIndicatorによる処理状態の可視化
- アイコンと色によるステータス表現（緑=アクティブ、黄=処理中、灰=待機）

## 効果

これらの改善により：

1. **初心者の理解促進**: 技術的な背景がなくても、システムの動作を理解できる
2. **探索の促進**: ヘルプを見ながら、安心して機能を試せる
3. **学習曲線の改善**: 段階的に深い理解へ進める
4. **信頼性の向上**: システムが何をしているかが明確になることで、信頼感が向上

## 技術的な実装詳細

### 新規コンポーネントAPI

```typescript
// Tooltip - 汎用ツールチップコンポーネント
<Tooltip content="説明文">
  <span>ホバー対象</span>
</Tooltip>

// HelpIcon - ヘルプアイコン（疑問符）
<HelpIcon content="この機能の説明..." />

// StatusIndicator - 状態表示
<StatusIndicator
  status="processing" // 'active' | 'idle' | 'processing'
  label="処理中"
/>
```

### 使用したアイコン

- `HelpCircle` (lucide-react): ヘルプアイコン
- 既存のアイコンと統一されたデザイン言語

### スタイリング

- Tailwind CSSによる一貫したデザイン
- 既存のカラースキームとの調和
- レスポンシブデザイン対応
